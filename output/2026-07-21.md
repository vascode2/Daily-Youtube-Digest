   - DeepSeek 및 TurboQuant 등의 사례를 통해 고성능 GPU와 HBM 메모리의 실제 필요량이 어떻게 변화하는지 분석합니다.

3. **NVIDIA와 미국 AI 업계의 긴장 및 투자자 우려**
   - 중국발 AI 효율성 혁명은 NVIDIA의 하드웨어 독점 체제에 경종을 울리며 미국 AI 기업들을 긴장시키고 있습니다.
   - 투자자들은 AI 효율성 극대화가 장기적으로 GPU 수요 감소로 이어질지, 아니면 새로운 수요를 창출할지 우려 섞인 시선으로 주시하고 있습니다.

4. **Anthropic 제기 논란과 기술적 쟁점**
   - Anthropic이 제기한 비정상적인 대량 호출 및 증류(Distillation) 논란과 관련하여 현재까지 확인된 사실들을 객관적으로 짚어봅니다.

## [Every Harness Will Become A Claw — Sam Bhagwat, Mastra](https://www.youtube.com/watch?v=8qWIPUia2O8)

**한 줄 인사이트**
💡 개발자 도구인 하네스(Harness)는 컨텍스트 엔지니어링과 코딩 에이전트의 결합을 넘어, 스스로 작동하고 협업하는 백그라운드 에이전트인 클로(Claw)로 진화할 것입니다.

**핵심 요약**
Mastra의 창업자 Sam Bhagwat은 단순한 컨텍스트 제공을 넘어선 코딩 에이전트의 미래 진화 방향을 제시합니다. 그는 개발 환경의 '하네스'가 외부 이벤트를 감지하고 사용자와 소통하는 자율적 에이전트인 '클로'로 확장되는 필연적인 과정을 설명합니다. 이를 통해 개발자가 자리를 비운 사이에도 스스로 작업을 수행하는 새로운 협업 워크플로우의 도래를 예고합니다.

1. **하네스의 정의와 진화**
   - 하네스는 단순한 컨텍스트 엔지니어링을 넘어 계획 모드, 병렬 서브에이전트, 백그라운드 작업 등이 결합된 강력한 개발자 경험(DX)을 제공합니다.
   - Claude Code와 같은 최신 코딩 에이전트의 등장으로 하네스는 개발자의 생산성을 극대화하는 핵심 도구로 자리 잡았습니다.

2. **하네스에서 클로(Claw)로의 확장**
   - 에이전트 팀들은 하네스에 외부 이벤트를 수신하고 사용자에게 알림을 보내는 '하트비트(Heartbeat)' 기능을 부여하여 자율성을 높이고 있습니다.
   - 이러한 변화를 통해 하네스는 사용자가 오프라인 상태일 때도 스스로 작동하며 협업 워크플로우에 깊숙이 관여하는 '클로'로 진화합니다.

3. **스타인버거의 법칙(Steinberger's Law)**
   - Sam Bhagwat은 모든 하네스가 결국 클로로 확장될 것이라는 '스타인버거의 법칙'을 제안하며 에이전트 기술의 필연적 발전 방향을 설명합니다.
   - Mastra와 같은 TypeScript 에이전트 프레임워크는 이러한 자율적이고 지속적인 에이전트 생태계를 구축하는 데 기여하고 있습니다.

## [HTML Is All Agents Need — James Russo, HeyGen](https://www.youtube.com/watch?v=Cz4v1WHVyZc)

**한 줄 인사이트**
💡 LLM 에이전트가 비디오를 생성할 때 복잡한 프레임워크 대신 웹 표준인 HTML, CSS, JavaScript를 사용하면 소형 모델로도 창의적이고 결정론적인 비디오 렌더링이 가능합니다.

**핵심 요약**
HeyGen의 소프트웨어 엔지니어 제임스 루소(James Russo)는 LLM을 활용해 비디오를 자동 생성하는 과정에서 겪은 시행착오와 해결책을 공유합니다. 기존의 무거운 프롬프트나 Remotion 같은 React 기반 프레임워크는 에이전트의 창의성을 제한했으나, LLM에게 가장 친숙한 HTML 표준을 도입하면서 돌파구를 찾았습니다. 저자는 소형 모델인 Gemini Flash에서도 안정적으로 작동하며 브라우저 엔진을 통해 결정론적인 MP4 비디오를 렌더링하는 'Hyperframes' 프레임워크의 개발 여정을 소개합니다.

1. **기존 비디오 생성 프레임워크의 한계와 HTML의 재발견**
   - React 기반의 Remotion 프레임워크는 에이전트에게 너무 많은 제약을 가해 결과물을 안전하지만 지루하게 만들었습니다.
   - 반면 LLM에게 가장 네이티브한 언어인 HTML, CSS, JavaScript를 사용하자 에이전트의 시각적 창의성이 다시 살아났습니다.

2. **소형 모델 Gemini Flash를 위한 Hyperframes 설계**
   - 가장 거대한 모델을 써야만 쓸만한 결과가 나온다면 프레임워크가 에이전트와 싸우고 있다는 증거이므로, 가볍고 빠른 Gemini Flash에 맞춰 프레임워크를 설계했습니다.
   - 소형 모델이 안정적으로 작성할 수 있는 구조로 프레임워크를 최적화하여 비용과 속도 문제를 동시에 해결했습니다.

3. **브라우저 환경에서 결정론적 MP4 렌더링 구현**
   - 브라우저의 이미지 디코더, 폰트 로더, 애니메이션 클록은 비동기로 작동하여 매번 동일한 픽셀을 렌더링하는 결정론적 비디오 생성에 걸림돌이 되었습니다.
   - 이를 해결하기 위해 브라우저 네이티브 기술인 Three.js와 WebGL을 활용하여 에이전트가 자유롭게 렌더링할 수 있는 환경을 구축했습니다.

4. **에이전트에게 제약 대신 '감각(Taste)'을 부여하는 스킬 시스템**
   - 단순히 에이전트의 행동을 제한하는 가이드라인 대신, 에이전트 루프와 자체 평가 시스템을 통해 좋은 비디오를 만드는 감각을 학습시켰습니다.
   - 이를 통해 인간이 개입하는 워크플로우 내에서 고품질의 비디오 스토리텔링을 효율적으로 자동화할 수 있게 되었습니다.
