## [SSD가 잘 팔리는데도 메모리 공급이 부족한 이유는? ... 예상치 못한 쇼티지의 원인](https://www.youtube.com/watch?v=bI8kiH8puxk)

**한 줄 인사이트**
💡 고성능 기업용 SSD의 대용량화로 인해 주소 매핑용 DDR4 DRAM 수요가 급증하는 반면, 제조사들의 감산으로 인해 예상치 못한 메모리 공급 부족과 가격 상승이 발생하고 있습니다.

**핵심 요약**
안될공학은 최근 기업용 SSD 시장의 활성화에도 불구하고 구형으로 여겨지던 DDR4 DRAM 가격이 반등하는 독특한 시장 현상을 분석합니다. SSD의 기술적 특성상 대용량화될수록 데이터 주소를 매핑하기 위한 전용 DRAM 요구량이 정비례하여 증가하는 구조적 원인을 설명합니다. 나아가 HBM과 DDR5로의 생산 전환으로 인한 DDR4 공급 부족이 맞물리며 발생한 메모리 시장의 쇼티지 현상을 진단합니다.

1. **SSD 데이터 매핑과 DRAM의 기술적 연계**
   - 낸드 플래시는 데이터를 기존 위치에 바로 덮어쓸 수 없어 데이터 위치가 계속 변하므로, 논리 주소와 실제 물리 위치를 매핑하는 주소 테이블 관리가 필수적입니다.
   - 고성능 기업용 SSD는 이 방대한 주소 테이블을 빠르게 탐색하기 위해 전용 DRAM을 사용하며, 통상 1TB의 낸드당 1GB의 DRAM이 필요하다는 업계 규칙이 적용됩니다.

2. **초고용량 SSD 트렌드로 인한 DRAM 탑재량 급증**
   - 기업용 SSD 용량이 30TB, 60TB를 넘어 최대 122TB까지 대형화되면서 단일 SSD 내에 탑재되어야 하는 DRAM의 용량도 비례하여 급격히 늘어났습니다.
   - 이는 단순히 SSD 판매 개수 증가를 넘어, 개별 제품당 소요되는 메모리 반도체의 절대적인 양을 크게 늘리는 결과를 초래했습니다.

3. **제조사들의 차세대 메모리 집중과 DDR4 공급 부족**
   - 주요 메모리 제조사들이 HBM과 DDR5 생산에 설비를 우선 배정하면서 구형 DDR4의 생산량과 공급을 급격히 축소시켰습니다.
   - 결과적으로 DDR4 수요는 대용량 SSD 등에서 지속되는 반면 공급이 더 빠르게 감소하면서, 시장에서 예상치 못한 가격 상승과 공급 부족 현상이 나타나고 있습니다.

## [Computer-Use 2.0: Agents Just Got Multi-Cursor — Francesco Bonacci, Cua](https://www.youtube.com/watch?v=ZSQb5fzRFPw)

**한 줄 인사이트**
💡 OS의 접근성 레이어를 직접 제어하는 cua driver는 하드웨어 커서 점유 없이 멀티 에이전트 구동을 지원하며, 벤치마크 통과율을 62%에서 80%로 높이고 토큰 소모는 34% 줄입니다.

**핵심 요약**
Francesco Bonacci는 기존 컴퓨터 사용(Computer Use) 모델의 한계를 극복하기 위해 OS 접근성 레이어와 직접 통신하는 'cua driver'와 검증 도구인 'CUABench'를 소개합니다. 그는 하드웨어 마우스 커서를 가로채는 대신 백그라운드 API를 활용하여 여러 에이전트가 동시에 독립적으로 작업을 수행할 수 있는 아키텍처를 제시합니다. 이를 통해 토큰 효율성을 극대화하는 동시에, 실제 회로 설계 소프트웨어와 같은 고난도 환경에서 에이전트 성능을 정밀하게 평가하고 개선하는 방법을 보여줍니다.

1. **OS 접근성 레이어를 활용한 백그라운드 제어**
   - Windows의 UI Automation, Linux의 AT SPI, macOS의 AX 등 OS 내부의 비공개 접근성 API를 통해 백그라운드 창에 직접 클릭과 키 입력을 전달합니다.
   - 사용자의 실제 마우스와 키보드가 움직이지 않으므로, 인간 사용자의 작업을 방해하지 않으면서 여러 에이전트가 동시에 서로 다른 앱을 제어할 수 있습니다.
2. **CUABench를 통한 에이전트 신뢰성 검증**
   - 5개 플랫폼과 42개 환경에 걸쳐 130개 이상의 검증 가능한 작업으로 구성된 CUABench를 구축하여 에이전트의 오작동과 편법(reward hacking)을 걸러냅니다.
   - 전체 화면 대신 단일 창만 관찰하는 방식을 채택하여 4K 벤치마크 통과율을 62%에서 80%로 끌어올렸으며, 토큰 사용량은 34% 절감했습니다.
3. **실제 산업용 소프트웨어 환경에서의 한계 극복**
   - Snorkel AI와 협력하여 실제 회로 설계 소프트웨어를 대상으로 한 전기 공학 벤치마크를 추가하여 에이전트의 한계를 시험했습니다.
   - 테스트 결과 가장 뛰어난 에이전트도 25개 회로 편집 작업 중 단 6개만 통과했으며, 빈 도면에서 새로 설계를 시작하는 작업에서는 모든 모델이 0%의 성공률을 기록했습니다.
4. **인프라 최적화와 모바일 확장 가능성**
   - 대규모 에이전트 구동을 위해 GPU 인프라를 최적화하는 Cua Fleet 아키텍처를 통해 시스템 효율성을 극대화합니다.
   - 데스크톱 환경을 넘어 모바일 및 안드로이드 지원을 포함한 다양한 플랫폼으로의 확장 계획을 논의하며 에이전트 생태계의 비전을 제시합니다.
