## [Claws Out: Securing and Building with OpenClaw - Nick Taylor, Pomerium](https://www.youtube.com/watch?v=xg1zNlzw7Jk)

**한 줄 인사이트**
💡 오픈소스 OpenClaw를 안전하게 사용하려면 Identity-Aware Proxy인 Pomerium과 신뢰할 수 있는 프록시 인증을 결합하여 외부 위협으로부터 워크스페이스를 보호해야 합니다.

**핵심 요약**
Pomerium의 Nick Taylor는 보안 장치 없이 OpenClaw를 실행할 때 발생할 수 있는 위험성을 경고하며 이를 안전하게 보호하는 아키텍처를 제안합니다. 그는 자신이 구축한 McClaw 인스턴스를 보호하기 위해 OpenClaw 프로젝트에 신뢰할 수 있는 프록시(trusted-proxy) 인증 모드를 직접 기여한 과정을 설명합니다. 나아가 브라우저 기반 파일 탐색기인 Clawspace를 라이브로 구축하며 안전한 환경에서 개발 도구를 확장하는 실무적인 방법을 시연합니다.

1. **OpenClaw 보안의 필요성과 위험성**
   - OpenClaw를 적절한 접근 제어 없이 실행하면 공격자가 워크스페이스에 무단으로 접근하여 악성 코드를 실행할 수 있는 심각한 보안 취약점이 발생합니다.
   - 개발자는 외부 노출을 최소화하고 신뢰할 수 있는 인증 계층을 전면에 배치하여 인프라와 데이터를 보호해야 합니다.
2. **Pomerium을 활용한 ID 인식 프록시 구축**
   - 오픈 코어 Identity-Aware Proxy인 Pomerium을 OpenClaw 앞단에 배치하여 세밀한 사용자 인증 및 권한 부여를 강제합니다.
   - 이를 통해 복잡한 VPN 설정 없이도 안전하게 외부에서 OpenClaw 인스턴스인 McClaw에 접근할 수 있는 환경을 제공합니다.
3. **Trusted-Proxy 인증 모드 기여**
   - Nick Taylor는 OpenClaw 오픈소스 프로젝트에 직접 'trusted-proxy' 인증 모드 기능을 추가하는 feat(gateway) 기여를 진행했습니다.
   - 이 기능은 프록시가 전달한 인증 헤더를 OpenClaw가 신뢰하도록 만들어 다중 보안 계층을 유연하게 통합할 수 있도록 돕습니다.
4. **Clawspace를 통한 워크스페이스 확장**
   - OpenClaw 워크스페이스 내의 파일을 브라우저에서 직접 탐색하고 편집할 수 있는 도구인 Clawspace(github.com/nickytonline/clawspace)를 소개합니다.
   - 라이브 코딩을 통해 보안이 확보된 OpenClaw 환경 위에서 어떻게 실시간으로 유용한 개발 도구를 빌드하고 연동하는지 증명합니다.
