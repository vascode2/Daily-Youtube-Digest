## [Why TTS Models Now Look Like LLMs — Samuel Humeau, Mistral](https://www.youtube.com/watch?v=3jGAU2sbAyY)

**한 줄 인사이트**
💡 TTS도 결국 토큰 시퀀스 문제로 수렴하면서, 음성 모델 평가·튠을 LLM 파이프라인과 동일한 도구체인으로 통합 설계해야 한다는 신호입니다.

**핵심 요약**

발표에서는 Mistral 내부에서 진행한 코덱 비트레이트 vs. 자연스러움 비교, 그리고 동일한 인스트럭션 튜닝 데이터로 음성/텍스트 모달리티를 동시에 미세조정한 사례를 보여줍니다. FAE 관점에서는 임베디드 디바이스의 음성 UX 제안 시 더 이상 별도의 음성 SDK 스택을 가정하지 말고, 클라이언트가 이미 운영 중인 LLM 인프라(GPU, 평가 셋, 안전 필터)를 재사용하는 통합 청사진을 먼저 제시하는 것이 효과적입니다.

## [Voice AI: when is the "Her" moment? — Neil Zeghidour, Gradium AI](https://www.youtube.com/watch?v=P_RI1kCkRbo)

**한 줄 인사이트**
💡 음성 에이전트의 "Her 순간"을 가르는 변수는 모델 IQ가 아니라 턴 종료 인식과 백채널링까지 포함하는 실시간 대화 루프 설계라는 점이 핵심입니다.

**핵심 요약**
풀듀플렉스 음성 대화가 가능한 시점을 모델 능력이 아니라 시스템 전체 지연(엔드포인트 검출 + 모델 응답 + 합성 시작)이 1초 이하로 떨어지는 시점으로 재정의합니다. Gradium에서는 이 목표를 위해 STT/LLM/TTS를 직렬로 쌓는 대신 음성 토큰을 직접 입출력하는 통합 모델로 단계 수를 줄이는 접근을 택했습니다.

데모에서는 사용자가 말 도중에 끼어들 때 모델이 즉시 "음, 알았어요" 같은 백채널을 내고 자연스럽게 화제를 전환하는 장면을 보여줍니다. FAE 관점에서는 콜센터·키오스크 PoC 제안 시 단일 STT 정확도 벤치마크가 아니라 평균 첫응답시간(TTFT 음성)과 인터럽션 회복 성공률 같은 대화형 KPI를 SOW에 명시하도록 가이드해야 고객 기대치를 정렬할 수 있습니다.

## [Microsoft doesn't understand FPS](https://www.youtube.com/watch?v=znRHdCDw9tQ)

**한 줄 인사이트**
💡 프레임 생성·업스케일링이 보이는 FPS 숫자를 부풀리는 동안 입력 지연은 그대로 남는다는 점을 마케팅과 분리해 평가해야 한다는 경고입니다.

**핵심 요약**
Microsoft가 Xbox/Windows에서 광고하는 "고FPS" 수치가 실제 게이머 체감과 다른 이유를 분석합니다. DLSS 3/FSR 3 류 프레임 생성 기술이 화면에 표시되는 프레임 수를 늘려도 입력 → 화면 반영까지의 모션 투 포톤 지연(motion-to-photon)은 줄지 않거나 오히려 늘어난다는 점이 핵심입니다.

발표자는 LDAT 같은 측정 장비로 측정한 실제 입력 지연 비교 그래프와, 프레임 생성을 켰을 때 마우스 에임이 미끄러지는 라이브 게임 플레이를 대조해 보여줍니다. FAE 관점에서는 산업용 디스플레이/HMI 장비 스펙 협상 시 단순 리프레시 레이트 숫자가 아니라 엔드 투 엔드 latency(터치 입력 → 픽셀 변경)를 명시하도록 고객을 가이드하면 같은 함정을 피할 수 있습니다.
