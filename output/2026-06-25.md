## [The Log Is The Agent - Ishaan Sehgal, Omnara](https://www.youtube.com/watch?v=UPwGaM2MKHY)

**한 줄 인사이트**
💡 AI 에이전트의 본질은 모델이나 툴이 아닌 '세션 로그(저장 파일)'에 있으며, 로그 중심의 인프라 구축이 에이전트의 지속성과 이식성을 결정하는 핵심이 될 것입니다.

**핵심 요약**
Omnara의 공동 창업자 Ishaan Sehgal은 비디오 게임의 세이브 파일 비유를 통해 AI 에이전트 인프라의 패러다임 전환을 제안합니다. 그는 LLM 모델과 툴이 점차 범용화됨에 따라 에이전트의 정체성과 상태를 유지하는 핵심은 결국 '세션 로그'가 될 것이라고 주장합니다. 이를 통해 에이전트의 지속성, 이식성, 분기 가능성을 확보하는 새로운 인프라 설계 방향을 제시합니다.

   - 비디오 게임 엘든 링(Elden Ring)의 캐릭터 본질이 콘솔 기기가 아닌 세이브 파일에 있듯, 에이전트의 정체성과 상태 역시 실행 엔진이 아닌 세션 로그에 저장됩니다.
   - 플레이스테이션이 고장 나도 클라우드에서 세이브 파일을 불러오면 되듯이, 에이전트 역시 로그만 있다면 어떤 환경에서든 이전 상태 그대로 복구할 수 있습니다.

   - LLM 엔진은 대체 가능해지고 툴은 단순한 API로 표준화되면서, 모델 제공업체나 프레임워크보다 세션 로그가 에이전트의 유일한 차별화 요소가 됩니다.
   - 10년 전 마틴 클레프만(Martin Kleppmann)이 데이터베이스를 추가 전용 로그(Append-only log)의 투영으로 정의했듯이, 에이전트 역시 세션 로그를 기본 단위로 재정의해야 합니다.

   - 실행 타임라인을 분기(Forkability)하여 병렬 탐색을 수행하거나, 모델과 런타임을 자유롭게 교체하는 이식성(Portability)을 실현할 수 있습니다.

4. **에이전트 소유권과 인프라의 미래 질문**
   - Omnara의 CEO Ishaan Sehgal은 모델 제공업체가 로그를 독점적으로 소유할 경우 에이전트 자체의 소유권도 그들에게 종속되는 문제를 지적합니다.
   - 미래의 에이전트 인프라는 특정 모델에 종속되지 않고 여러 머신과 런타임 간에 자유롭게 이동할 수 있는 독립적이고 이식 가능한 로그 시스템을 지향해야 합니다.
