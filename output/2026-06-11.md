## Why Can't Anyone Answer Questions About the Business? — Garrett Galow, WorkOS

**핵심 요약**
WorkOS의 Garrett Galow는 비즈니스 질문에 대한 SQL 기반 답변 과정이 비효율적인 반복 루프에 갇혀 있음을 지적합니다. 그는 이러한 문제를 해결하기 위해 엔지니어의 개입 없이 자연어로 데이터에 접근하고 재사용 가능한 위젯을 생성할 수 있는 내부 워크스페이스 'Studio'를 구축했습니다. Studio는 Snowflake, Linear, Notion 등 다양한 데이터 소스를 통합하여 누구나 쉽게 비즈니스 질문에 대한 답을 얻을 수 있도록 돕습니다.

1.  **SQL 질문 루프의 비효율성**
    -   비즈니스 질문에 SQL이 필요할 때마다 질문 설명, 엔지니어 대기, 답변 획득, 추가 조인 요청, Slack 공유의 반복적인 과정을 거쳐 시간과 자원이 낭비됩니다.
2.  **Studio: 자연어 기반 데이터 접근 솔루션**
    -   WorkOS의 Garrett Galow는 이러한 루프를 끊기 위해 'Studio'를 개발했습니다. 이는 누구나 Snowflake, Linear, Notion 등의 데이터 소스에 자연어로 질문하고 답변을 얻을 수 있는 내부 워크스페이스입니다.
    -   Studio는 요청 제출 없이도 답변이나 재사용 가능한 위젯을 제공하여 데이터 접근성을 크게 향상시킵니다.
3.  **재사용 가능한 위젯의 핵심 기능**
    -   LLM이 선언적 JavaScript로 위젯을 한 번 작성하면, 이는 기본 데이터 소스를 직접 호출하여 이후 실행이 확정적이고 비용 효율적입니다.
    -   이 위젯은 스키마 컨텍스트를 도구 호출 시점에만 주입하는 'Preflight sequencing', 모델이 WorkOS에 대한 자체 지식보다 1차 소스를 신뢰하도록 하는 'Layering rule', 그리고 생성된 Snowflake 쿼리를 위젯에 하드코딩하기 전에 검증하는 'Query validation'을 통해 신뢰성을 확보했습니다.

## [The agent-ready web: Simplify user actions with WebMCP — Tara Agyemang, Google](https://www.youtube.com/watch?v=ghJmWQCIHRM)

**한 줄 인사이트**
💡 WebMCP는 AI 에이전트가 웹사이트와 상호작용하는 복잡한 과정을 구조화된 도구로 대체하여 사용자 작업을 간소화하는 새로운 웹 표준입니다.

**핵심 요약**
구글 크롬 팀의 Tara Agyemang은 현재 AI 에이전트가 웹사이트에서 사용자 작업을 수행하는 방식이 비효율적이고 오류 발생 가능성이 높다고 지적합니다. 그녀는 DOM 분석, 스크린샷, 픽셀 좌표 계산 등 복잡한 과정을 구조화된 도구로 대체하는 새로운 웹 표준인 WebMCP를 소개합니다. WebMCP는 에이전트가 웹사이트의 기능을 추측하는 대신, 명명되고 유형이 지정된 작업 메뉴를 직접 호출할 수 있도록 하여 상호작용을 간소화합니다.

1.  **기존 AI 에이전트의 웹 상호작용 문제점**
    *   AI 에이전트가 웹에서 사용자 작업을 수행하려면 전체 DOM, 접근성 트리, 스크린샷, 픽셀 좌표 계산 등 복잡하고 오류 발생 가능성이 높은 과정을 거쳐야 합니다.
    *   예를 들어, 콘서트 티켓 2장을 구매하는 과정에서 광고 로딩으로 레이아웃이 변경되면 클릭이 빗나갈 수 있는 등 비효율성이 큽니다.
2.  **WebMCP: 구조화된 도구로의 전환**
    *   WebMCP는 에이전트가 웹사이트의 기능을 추측하는 대신, 명명되고 유형이 지정되며 설명된 작업 메뉴를 직접 호출할 수 있도록 하는 새로운 웹 표준을 제안합니다.
    *   이를 통해 에이전트는 복잡한 DOM 분석 없이도 웹사이트와 효율적으로 상호작용할 수 있습니다.
3.  **WebMCP의 두 가지 구현 경로**
    *   **선언적 API:** 기존 폼에 HTML 속성을 추가하여 브라우저가 JSON 스키마를 자동으로 생성하게 합니다.
    *   **명령형 API:** 복잡한 다단계 흐름을 위해 JavaScript에서 사용자 정의 도구를 등록하고, 일반 DOM 코드를 실행하여 에이전트에 상태를 반환합니다.
4.  **라이브 데모 및 현재 개발 현황**
    *   라이브 데모에서는 WebMCP를 사용하여 콘서트 티켓 구매를 단 세 번의 도구 호출(이름으로 검색, 콘서트 페이지 열기, 수량 및 섹션으로 구매 호출)로 완료하는 과정을 시연했습니다.
    *   WebMCP는 현재 Chrome 146에서 실험적 기능으로 초기 미리보기가 제공되며, 자체 웹사이트 테스트를 위한 eval CLI 및 인스펙터 확장 프로그램도 이용 가능합니다.
