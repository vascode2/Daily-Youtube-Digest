## [20 days of compute vs 7 hours: rethinking what state-of-the-art means — Bertrand Charpentier, Pruna](https://www.youtube.com/watch?v=hqHC6Z_lXyo)

**한 줄 인사이트**
💡 진정한 SOTA(최첨단) 모델은 단순히 성능 점수만 높은 모델이 아니라, 파레토 최적화(Pareto front)를 통해 비용과 지연 시간 대비 최고의 효율성을 증명하는 모델입니다.

**핵심 요약**
Pruna AI의 공동 창업자이자 수석 과학자인 Bertrand Charpentier는 기존 벤치마크 평가 방식의 엄청난 컴퓨팅 비용과 비효율성을 지적하며 '최첨단(SOTA)'의 정의를 재정립해야 한다고 주장합니다. 그는 리더보드 순위의 맹점을 폭로하고 품질과 효율성을 동시에 고려하는 파레토 전선(Pareto front) 모델을 제시합니다. 이를 통해 거대 기초 모델 대신 특정 작업에 최적화된 작고 효율적인 전문 모델을 선택하는 것이 실제 비즈니스 환경에서 왜 더 유리한지 논증합니다.

1. **기존 이미지 생성 모델 평가의 엄청난 비용 문제**
   - Design Arena 방식의 26,000회 배틀 평가는 20일의 컴퓨팅 시간과 5,000달러의 비용, 그리고 마라톤 400회 분량의 에너지를 소모합니다.
   - 반면 Pruna AI의 압축 모델을 사용하면 동일한 평가를 단 7시간 만에 완료하고 비용도 265달러 수준으로 대폭 절감할 수 있습니다.
2. **단일 리더보드 순위가 가진 한계와 왜곡**
   - 동일한 모델이 평가 플랫폼에 따라 5위에서 10위까지 순위가 요동치며, 단일 리더보드 점수만으로는 모델의 실제 성능을 신뢰하기 어렵습니다.
   - 상위권 모델조차 일대일 비교 배틀에서 약 40%의 확률로 패배하므로, 1위 모델이 실제 개별 사용 사례의 절반 가까이에서는 오답이 될 수 있습니다.
3. **품질과 효율성을 동시에 극대화하는 파레토 최적화**
   - 성능 품질과 지연 시간 또는 비용을 축으로 하는 파레토 전선(Pareto front)을 구축하면 최적의 모델 군을 명확히 식별할 수 있습니다.
   - 이 전선 상의 모델들은 품질 점수는 매우 유사하지만, 운영 효율성 측면에서는 최대 20배까지 차이가 날 수 있습니다.
4. **거대 모델보다 작고 전문화된 모델의 가치**
   - 파레토 전선 분석을 적용하면 값비싼 거대 기초 모델보다 특정 도메인에 특화된 작고 가벼운 전문 모델들이 표면 위로 드러납니다.
   - 효율성은 단순한 부가 요소가 아니라 최첨단 기술을 정의하는 핵심 차원이며, 실제 상용화 단계에서 가장 먼저 고려되어야 합니다.

## [What if the network was the sandbox? — Remy Guercio, Tailscale](https://www.youtube.com/watch?v=BM2JX9hqsVQ)

**한 줄 인사이트**
💡 샌드박스 내부에 API 키를 두는 기존 방식 대신, 네트워크 레이어에서 신원을 검증하고 LLM 호출을 통제함으로써 에이전트의 자격 증명 탈취 및 오용 위험을 근본적으로 차단할 수 있습니다.

**핵심 요약**
Tailscale의 Remy Guercio는 기존의 샌드박싱 방식이 실행 격리와 액세스 제어라는 두 가지 서로 다른 문제를 혼동하고 있다고 지적합니다. 그는 에이전트가 자격 증명을 직접 보유함으로써 발생하는 보안 취약점을 해결하기 위해 WireGuard ID 네트워크 기반의 LLM 게이트웨이인 Aperture를 대안으로 제시합니다. 이를 통해 컨테이너 내부의 별도 도구 없이도 네트워크 레이어에서 모든 LLM 호출, bash 명령, MCP 요청을 안전하게 모니터링하고 제어하는 방법을 설명합니다.

1. **기존 샌드박싱의 보안 한계와 자격 증명 노출 문제**
   - 기존의 샌드박스 방식은 에이전트가 실행되는 환경 내부에 API 키를 직접 배치하므로, 에이전트가 이를 외부로 유출하거나 오용할 위험에 노출되어 있습니다.
   - 실행 환경을 완벽하게 격리하더라도 에이전트가 여전히 남용 가능한 자격 증명을 보유하고 있다면 근본적인 액세스 제어 문제를 해결할 수 없습니다.

2. **Tailscale WireGuard 기반의 LLM 게이트웨이, Aperture**
   - Aperture는 모든 연결에 사용자, 태그, 그룹 등의 검증된 신원 정보를 실어 보내며, 에이전트에게 실제 키 대신 플레이스홀더(placeholder)를 제공합니다.
   - 에이전트가 실제 자격 증명을 직접 가지지 않기 때문에 외부로 유출할 수 있는 물리적인 키 자체가 존재하지 않게 됩니다.

3. **네트워크 레이어 통제를 통한 무설치 모니터링 구현**
   - 모든 LLM 호출이 네트워크 레이어를 통과하도록 강제함으로써, 컨테이너 내부에 별도의 모니터링 도구를 설치하지 않고도 모든 활동을 감시할 수 있습니다.
   - 이를 통해 Aperture는 에이전트가 수행하는 모든 도구 호출, bash 명령, MCP(Model Context Protocol) 요청을 투명하게 모니터링하고 기록합니다.

   - 이미지를 활용한 시각적 스펙은 수많은 텍스트 프롬프트를 대체하며 에이전트의 기획 및 설계 제약 조건을 효과적으로 해결해 줍니다.
