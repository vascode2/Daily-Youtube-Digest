## [Your Agent Is an Infinite Canvas — RL Nabors, Dressed for Space](https://www.youtube.com/watch?v=LMbeDEQO6QM)

**한 줄 인사이트**
💡 에이전트의 미래는 단순한 채팅창을 넘어, 웹 표준 기술을 활용해 실시간으로 상호작용하는 '무한한 캔버스'로 진화할 것입니다.

**핵심 요약**
웹 애니메이션 전문가이자 개발자인 RL 네이버스(RL Nabors)는 텍스트 기반의 채팅 인터페이스가 AI 에이전트의 최종 목적지가 될 수 없다고 주장하며, 웹 플랫폼을 활용한 새로운 UI 패러다임을 제시합니다. 그녀는 Claude 내부에서 직접 작동하는 웹툰 리더기 데모를 통해, MCP(Model Context Protocol)와 WebMCP 기술이 어떻게 에이전트와 웹을 결합하는지 보여줍니다. 이를 통해 개발자 중심의 터미널 같은 채팅창에서 벗어나, 사용자가 웹 기술로 구현된 풍부한 인터랙티브 화면을 에이전트 안에서 직접 경험하는 미래를 제안합니다.

1. **채팅 UI의 한계와 터미널 비유**
   - 현재의 채팅창 중심 에이전트 UI는 과거 PC 시절의 터미널(CLI)과 같아서 개발자에게는 유용하지만 대중적인 인터페이스가 되기에는 한계가 있습니다.
   - 결국 에이전트 소프트웨어도 터미널에서 아이폰(GUI)으로 진화했듯, 텍스트 응답을 넘어 시각적이고 직관적인 인터랙티브 화면으로 전환되어야 합니다.
2. **MCP 앱을 통한 인터랙티브 화면 구현**
   - MCP(Model Context Protocol) 앱은 HTML, CSS, JavaScript를 단일 파일로 묶어 에이전트 인터페이스 내의 iframe에서 렌더링하는 방식을 취합니다.
   - 이를 통해 에이전트의 도구 응답을 단순한 텍스트가 아닌, 사용자가 직접 조작하고 탐색할 수 있는 실시간 웹 화면으로 전환할 수 있습니다.
3. **브라우저 에이전트를 위한 WebMCP의 도입**
   - WebMCP는 기존 웹 페이지의 폼(Form) 요소에 도구 이름과 설명 속성을 추가하여 브라우저 에이전트가 사이트의 기능을 직접 호출할 수 있게 합니다.
   - 이 방식은 에이전트가 화면을 캡처해 파싱하거나 복잡한 DOM 트리를 탐색할 필요 없이, 웹사이트의 API와 기능을 선언적으로 직접 제어하도록 돕습니다.
4. **웹 플랫폼이라는 무한한 캔버스의 활용**
   - 웹 표준 기술은 이미 음성 인식, 애니메이션 등 강력한 네이티브 API를 갖추고 있어 에이전트를 위한 완벽한 '무한한 캔버스' 역할을 수행합니다.
   - 개발자는 새로운 UI 프레임워크를 배울 필요 없이 기존의 웹 기술을 그대로 활용하여 에이전트 전용 사용자 경험을 설계할 수 있습니다.

3. **인간 중심 도구인 GitHub의 한계**

4. **Ona가 제시하는 VM 격리 기반 조율 솔루션**
