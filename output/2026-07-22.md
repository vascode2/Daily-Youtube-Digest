monday.com의 오므리 브루킴(Omri Bruchim)과 토머 아스트(Tomer Ast)는 오늘날의 AI가 방대한 사내 데이터를 모두 가지고도 엉뚱한 조언을 하는 이유가 검색 능력 부족이 아니라 기록(Record) 중심 구조의 한계 때문이라고 진단한다. 이들은 데이터를 의미 있는 맥락으로 전환하기 위해 느린 엔진과 빠른 엔진을 결합한 람다 아키텍처 기반의 세계 모델을 구축하여 에이전트의 판단력을 높이는 방법을 설명한다.

2. **맥락 구축을 위한 두 개의 엔진**
   - monday.com은 사전에 컨텍스트 레이어를 구축하기 위해 수주의 활동을 마이닝하여 지속적인 프로필을 만드는 느린 엔진을 활용한다.
   - 동시에 최근 며칠간의 긴급한 이슈와 협업 관계를 파악하는 빠른 엔진을 결합하여 개인과 하루의 흐름을 동시에 이해한다.

3. **뇌 구조와 람다 아키텍처의 결합**
   - 이 두 엔진의 분리는 뇌의 해마와 신피질 구조뿐만 아니라 데이터 시스템의 람다 아키텍처와도 유사한 구조를 띤다.

## [당신의 해자는 데이터 모델입니다 — Mike Phipps, Gates Foundation](https://www.youtube.com/watch?v=jt1Pbr_n6oU)

**한 줄 인사이트**
💡 AI 모델과 프레임워크가 빠르게 표준화되는 시대에 진정한 경쟁 우위는 조직의 암묵지와 질문 처리 방식을 담아낸 고유의 데이터 모델에 있습니다.

**핵심 요약**
마이크 피프스(Mike Phipps)는 게이츠 재단에서 25년간 축적된 방대한 보조금 데이터를 Neo4j 지식 그래프로 구축하고 Claude와 연동한 사례를 통해 에이전트 중심 아키텍처의 핵심을 설명합니다. 대시보드가 아닌 에이전트를 위해 데이터를 구조화하고 복잡한 계층 구조를 연결함으로써 조직 고유의 맥락을 보존하는 방법을 다룹니다.

1. **AI 시대의 진정한 해자**
   - 모델, 프레임워크, 프론트엔드가 모두 상품화되는 상황에서 조직의 데이터 모델과 질문에 답하는 암묵적 지식이 유일하게 지속 가능한 경쟁 우위가 됩니다.
   - 게이츠 재단은 25년간의 보조금 지급 내역과 4,000명의 인력을 아우르는 데이터를 단일 시스템으로 통합했습니다.

2. **에이전트를 위한 지식 그래프 구축**
   - 1년 예산이 7빌러블 달러에 달하고 2,000개의 보조금이 얽힌 방대한 운영 데이터를 에이전트 검색에 최적화된 구조로 재설계했습니다.
   - 비정형 문서를 청킹하고 태깅하여 정형 엔티티와 매핑함으로써, 에이전트가 복잡한 교차 시스템 질문을 단 한 번의 그래프 쿼리로 해결할 수 있게 합니다.

3. **조직 사일로 극복과 MCP 연동**
   - 자금 지원 및 관리 관점의 계층 구조와 인력 조직도를 통합하여 분산된 시스템을 유기적으로 연결했습니다.
   - 구축된 Neo4j 지식 그래프를 단 하나의 MCP(Model Context Protocol) 서버를 통해 Claude에 제공하여 조직 맞춤형 답변을 도출합니다.

4. **검색 평가와 피드백 루프**
   - 아키텍처의 신뢰성을 유지하기 위해 엄격한 검색 평가(retrieval evals) 프로세스를 도입하고 지속적인 피드백 루프를 운영합니다.
   - 상위 레이어가 아닌 데이터 모델 자체에 집중하는 소규모 팀의 노력이 어떻게 장기적인 복리 효과를 만드는지 보여줍니다.

## [Active Graph Agent Runtime (BabyAGI 4) — Yohei Nakajima, Untapped Capital](https://www.youtube.com/watch?v=khVX_BUnEwU)

**한 줄 인사이트**
💡 BabyAGI의 창시자 요헤이 나카지마는 LLM 중심 구조에서 벗어나 불변 이벤트 로그를 기반으로 에이전트의 상태를 관리하는 ActiveGraph 런타임을 소개합니다.

**핵심 요약**
영상의 발표자인 요헤이 나카지마는 500개 문항 평가 중 API 키가 중단되었을 때 처음부터 다시 시작하는 대신 로그를 통해 정확히 중단된 지점부터 복구한 경험을 공유합니다. 그는 기존의 에이전트 아키텍처를 뒤집어 로그 자체가 에이전트가 되는 구조를 제안하며, 이를 통해 시스템의 안정성과 자기 개선 능력을 극대화할 수 있음을 설명합니다.

1. **로그 중심의 아키텍처 설계**
   - BabyAGI의 창시자인 요헤이 나카지마는 LLM 주변에 메모리와 로깅을 덧붙이는 방식 대신, 불변 이벤트 로그를 중심으로 에이전트를 구축하는 방식을 제안합니다.
   - 모든 액션과 상태 변화가 타입이 지정된 단일 로그로 평탄화되어, 추가 비용 없이 에이전트의 리플레이, 롤백, 포크가 가능해집니다.

2. **행동, 정책 및 모듈식 팩**
   - 로그 위에는 그래프 변화에 반응하고 이벤트를 발생하는 행동, 인간의 개입이나 모순 검증 여부를 결정하는 정책이 배치됩니다.
   - 메모리, 도구, 채팅을 위한 교체 가능한 팩을 지원하며, LLM들끼리 직접 통신하지 않고 1970년대 블랙보드 시스템과 Kafka에서 차용한 공유 상태만을 참조합니다.

3. **자기 개선과 실험적 자율성**
   - 에이전트를 포크하고 패치를 제안한 뒤 샌드박스 테스트를 거쳐 정확도가 상승할 때만 유지하는 자기 개선 루프가 작동합니다.
   - ActiveGraph Lab은 블로그 포스트를 읽고 자체 실험을 실행하며, 실제 코드 버그를 스스로 찾아 PR을 오픈하는 자율성을 보여줍니다.
