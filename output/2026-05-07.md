## [Playground in Prod: Optimising Agents in Production Environments — Samuel Colvin, Pydantic](https://www.youtube.com/watch?v=A48uhxfxbsM)

**핵심 요약**
> Samuel Colvin은 Pydantic AI와 Logfire를 기반으로, 에이전트를 프로덕션에서 운용할 때 성능·비용·정확도를 함께 관리하는 방법을 설명합니다. 발표의 중심은 "좋은 프롬프트를 한 번 작성"하는 접근보다, 평가 루프(evals)와 변수 관리(managed variables)를 결합해 지속적으로 에이전트를 개선하는 운영 체계에 있습니다.
>
> 예를 들어 Jeppa 최적화 데모에서는 단순 프롬프트 대비 개선된 프롬프트가 더 높은 정확도를 보이는 과정을 비교 실행으로 보여주고, 이어서 managed variables를 통해 서버 재배포 없이 지시문·모델 설정을 바꾸는 사례를 시연합니다. 또한 공개 데이터와 비공개 데이터에서 평가 설계가 달라진다는 점, 그리고 과적합을 막기 위해 학습/검증 데이터를 분리해야 한다는 실무 기준도 함께 제시합니다.

**주요 타임라인**
- [00:00:14] 발표자 소개와 Pydantic/Pydantic AI/Logfire 역할 정리
- [00:08:00] Jeppa 기반 프롬프트 최적화 원리(후보 생성-평가-반복)
- [00:18:00] eval 실행과 프롬프트 성능 비교 방법 시연
- [00:43:00] managed variables로 런타임 설정을 교체하는 운영 패턴 설명
- [01:03:00] 재배포 없이 응답 동작을 바꾸는 실제 데모와 운영 시사점

- [00:13:52] 백엔드 지속 작업과 운영 경계를 먼저 고정하는 접근 소개
- [00:38:09] 코드 패턴을 표준화해 반복 오류를 줄이는 실전 설명

## [Everything You Need To Know About Agent Observability — Danny Gollapalli and Ben Hylak, Raindrop](https://www.youtube.com/watch?v=dcBGcM6ZwKk)

**핵심 요약**
> 이 발표는 에이전트 관측성을 로그 수집 수준으로 보지 않고, 실행 단계별 원인 분석과 자동 진단까지 연결하는 운영 체계로 설명합니다. 핵심은 에이전트 실패를 "모델 품질" 하나로 환원하지 않고, 호출 경로, 질의 패턴, 도구 사용 흐름을 함께 추적해 재현 가능한 문제로 바꾸는 데 있습니다.
>
> 예를 들어 발표자는 query API 호출 흐름을 분해해 어떤 구간에서 병목과 실패가 반복되는지 확인하는 사례를 보여주고, self-diagnostic 트리거를 통해 장애 상황에서 에이전트가 스스로 상태를 점검하도록 구성하는 접근을 제시합니다. 또한 운영팀이 며칠 단위 실험으로 베이스라인을 만들고 점진적으로 정책을 조정하는 방식까지 소개해, 관측성을 제품 운영 루틴으로 내재화하는 방법을 구체화합니다.

**주요 타임라인**
- [00:00:14] 에이전트 관측성의 범위와 문제 정의
- [00:02:22] 메모리/호출 경로 관측 포인트 소개
- [00:14:55] query API 기반 실행 흐름 분석 사례
- [00:22:06] self-diagnostic 트리거를 활용한 자동 점검 접근
- [00:33:15] 운영 기준선 수립과 실험 루프 설계
- [00:46:28] 현장 적용 질의응답과 단계적 도입 전략
