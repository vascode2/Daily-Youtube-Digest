1. **Grill me 스킬을 통한 기획의 압박 테스트**
   - 단 4줄의 프롬프트로 AI를 면접관으로 세워 기획의 빈틈을 결정 트리 구조로 집요하게 질문하며 보완하도록 만듭니다.
   - 핀테크 앱 개발 시 필수적인 보안, 프라이버시, 외부 API 전송 전 마스킹 여부 등 12개의 핵심 분기 질문을 추천 답안과 함께 제시해 결정 피로를 줄여줍니다.

2. **기획 문서(PRD)의 한계와 수동 설계 보완**
   - Grill me 스킬은 비전 수립과 합의 도출에는 강력하지만, 구체적인 구현 디테일이나 아키텍처 설계는 다소 빈약할 수 있습니다.
   - AI가 제안한 결과물과 수동 설계를 상호 비교하고 충돌하는 결정을 해소하여 최종적으로 단단한 PRD 문서를 확정해야 합니다.

3. **Claude와 Codex의 적재적소 모델 분업**
   - 전체적인 개발 계획과 설계는 추론 능력이 뛰어난 Claude에게 위임하고, 실제 코드는 Codex(GPT-5.5)가 작성하도록 역할을 분담합니다.
   - 각 AI 모델의 강점을 극대화하는 적절한 분업 시스템을 구축하는 것이 현대 에이전틱 엔지니어링의 핵심 역량입니다.

4. **하네스 프레임워크를 통한 자율 구현 루프**
   - 진입점인 `/harness` 스킬을 통해 전체 스펙을 단계별(phase) 및 세부 작업(step)으로 잘게 쪼갠 뒤 자율 루프를 실행합니다.
   - `executor.py`가 계획(Plan), 실행(Execute), 검증(Verify) 루프를 스스로 반복하며, 자기완결성을 가진 실행 가능 커맨드로 결과물을 검증합니다.

5. **3시간 반 만에 완성된 5단계 SaaS MVP**
   - 개발자가 자리를 비운 3시간 반 동안 AI 에이전트가 거의 개입 없이 기초 설계부터 코어 루프, API, 결제, 랜딩 페이지까지 5개 단계를 혼자 구현했습니다.

## [메타가 공개한 DRAM 재활용 칩... 메모리 수요 감소할까? HBM 다음은 CXL?](https://www.youtube.com/watch?v=PDPcZ8ERWe4)

**한 줄 인사이트**
💡 HBM이 속도 경쟁이라면 CXL은 메모리를 얼마나 효율적으로 활용하느냐의 경쟁이며, 메타는 은퇴한 서버의 DDR4를 CXL로 재활용해 서버 수와 처리 시간을 모두 줄이는 데 성공했습니다.

**핵심 요약**
안될공학은 메타가 은퇴한 서버의 DDR4 메모리를 최신 서버에 재활용하기 위해 자체 설계한 CXL Type-3 메모리 확장 칩 'Vistara'의 실체와 성능을 분석합니다. 저자들은 캐시 및 머신러닝 파라미터 서버 사례를 통해 상대적으로 느린 DDR4를 추가했음에도 오히려 전체 서버 수와 처리 시간이 감소한 기술적 비결을 규명합니다. 나아가 CXL이 HBM을 대체하는 기술이 아니라 메모리 효율성을 극대화하는 솔루션임을 밝히며 삼성전자, SK하이닉스, 파네시아 등 국내외 생태계의 역할을 조망합니다.

1. **메타의 CXL 기반 메모리 재활용 칩 'Vistara' 도입**
   - 메타는 은퇴한 서버에서 회수한 DDR4 메모리를 최신 서버에서 재사용하기 위해 CXL Type-3 메모리 확장 칩인 Vistara를 직접 설계하고 실제 서비스 환경에 배포했습니다.
   - 논문 분석을 통해 Vistara가 단순한 운영 방식의 개선이 아니라 실제 작동하는 반도체 칩임을 검증하고, 로컬 DDR5와 CXL DDR4 간의 대역폭 차이를 확인했습니다.

2. **느린 메모리 추가에도 서버 효율이 향상된 비결**
   - 캐시 및 머신러닝(ML) 파라미터 서버 분석 결과, 상대적으로 느린 DDR4를 CXL로 연결해 용량을 확장했음에도 불구하고 필요한 서버 수와 데이터 처리 시간이 오히려 감소했습니다.
   - 이는 메모리 계층화(Memory Tiering)를 통해 자주 사용하지 않는 데이터를 CXL 영역으로 배치함으로써 고가의 고성능 메모리를 효율적으로 비워두었기 때문입니다.

3. **CXL과 HBM의 본질적인 기술적 차이**
   - HBM이 극도의 대역폭과 속도를 추구하는 기술이라면, CXL은 메모리를 직접 연결해 확장하거나 여러 서버가 공유하는 메모리 풀을 구축하여 자원 활용도를 극대화하는 기술입니다.
   - 따라서 CXL은 HBM을 직접 대체하는 기술이라기보다는, 데이터센터의 DRAM 공급 부족과 비용 문제를 해결하기 위한 상호보완적 솔루션에 가깝습니다.

4. **CXL 생태계의 과제와 국내 기업들의 역할**
   - 메모리 계층이 늘어남에 따라 운영체제(OS)와 AI 런타임의 관리 복잡도가 급격히 증가하므로, 이를 효율적으로 제어할 소프트웨어 기술이 핵심 과제로 부상하고 있습니다.
   - CXL 생태계 활성화를 위해 삼성전자와 SK하이닉스는 하드웨어 공급을 주도하고 있으며, 파네시아(Panesia) 등은 CXL 컨트롤러 및 IP 기술 분야에서 중요한 역할을 담당하고 있습니다.
